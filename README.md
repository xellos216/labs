# Purpose

- **This repository is a centralized space for managing long-term technical learning records.**

- **Directories are organized by roadmap lifecycle, and learning materials are stored primarily in Markdown format.**

- **Centralized management of personal learning records**

- **Easy knowledge reuse and review**

## Repository Structure

- `active/` — roadmaps with ongoing sessions, exercises, or near-term expansion
- `backlog/` — planned or paused roadmaps
- `archive/` — roadmaps whose planned learning sessions are complete and remain available for review and reference
- `docs/labs/` — project rules, templates, source policy, and roadmap navigation

Roadmap lifecycle status describes where a track belongs now. Session archives remain inside their roadmap directory so the roadmap and its learning history move together.

See [`docs/labs/ROADMAP_INDEX.md`](docs/labs/ROADMAP_INDEX.md) for the current roadmap inventory and navigation rules.
