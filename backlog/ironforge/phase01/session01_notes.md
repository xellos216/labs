# Phase 01 - Session 01 Notes

## Core Principle

Hardware exists to serve workloads.

A computer should be designed according to its purpose.

---

## Workload-Driven Design

Examples:

- Gaming PC → GPU-focused
- NAS → Storage-focused
- Router → Network-focused
- Virtualization Host → RAM-focused
- Database Server → Storage IOPS-focused

---

## Systems Thinking

Bad approach:

Hardware → Find a purpose

Good approach:

Purpose → Workload → Hardware

---

## Linux Connection

Linux can run on many different types of systems because Linux adapts to the workload.

Examples:

- Router
- Server
- NAS
- Workstation
- Embedded device

---

## Networking Connection

Routers and firewalls are specialized computers.

Networking devices are built around packet-processing workloads.

---

## Security Connection

Security consumes resources.

Examples:

- Encryption → CPU
- Logging → Storage
- Monitoring → RAM
- Packet Inspection → CPU + RAM

---

## Key Insight

Purpose determines workload.

Workload determines hardware.

Hardware should be selected to satisfy workload requirements.
